Nmap Port Scanning Methodology (Commented Cheatsheet)

Feb 16, 2026 in METHODOLOGY
nmap port scanning
7 min read

Introduction

Nmap is the reference tool for port scanning during the enumeration phase. However, due to its many features, the absence of a clear methodology can lead to missing or misinterpreting information, thereby impacting the quality of subsequent analysis and exploitation phases.

This article, presented as a commented cheatsheet, proposes a structured port scanning methodology based on Nmap’s different phases. The objective is to present an approach applicable in real-world scenarios and to understand which commands to use and in what context.

The first part presents briefly how Nmap works and the challenges associated with understanding it. The second part presents the proposed methodology.

Nmap Phases: Overview and Challenges

Nmap operates through several distinct phases, each serving a specific role in the enumeration process.

For example, running nmap <target> triggers basic phases such as host discovery and port scanning1.

Conversely, running nmap -A <target>, enables additional phases, including service version detection (-sV), operating system detection (-O), traceroute (--traceroute), and execution of default NSE scripts (-sC)2.

Although both approaches may be suitable in some situations, they present certain limitations. In the first case, a default scan may miss essential information, such as service versions or vulnerability-related details. In the second case, using -A may result in longer, noisier scans that are more difficult to analyze and debug.

To address these limitations, the methodology presented in the next section is based on Nmap’s different phases, used as a structured foundation for the scanning process. The main steps are:

  1. Target enumeration
  2. Host discovery
  3. Port scanning (TCP and UDP)
  4. Advanced ports scan and other useful options

1. Target enumeration

  • List target IP addresses without scanning them : nmap -sL <target>
    • üí° Inputs: single IP/domain, range (10.10.10.15-20), or subnet (10.10.10.0/24)
    • üí° Reverse DNS resolution is enabled by default ‚Üí use -n to disable it (reduces scan time)

2. Host discovery

Host discovery is used to determine if hosts are active. There are 3 methods based on the OSI model:

üí° By default, if a host is active, Nmap performs reverse DNS resolution

  • --dns-servers <DNS_SERVER> : specifies a DNS server
  • -R: forces reverse DNS resolution for offline hosts

1. Using ARP requests (layer 2)

  • Using nmap : sudo nmap -PR -sn <target>
    • üí° Reminder: <target> can be IP address(es), range or subnet
    • -sn: host discovery only (no port scan)

2. Using ICMP requests (layer 3)

  • ICMP Echo Request (Type 8): sudo nmap -PE -sn <target>
    • ‚ùó May be ignored by Windows hosts. Alternatives:
      • Timestamp request (ICMP Type 13): sudo nmap -PP -sn <target>
      • Address mask request (ICMP Type 17): sudo nmap -PM -sn <target>

3. Using TCP and UDP requests (layer 4)

  • TCP SYN ping: sudo nmap -PS -sn <target>
    • Requires root privileges to send forged RST packets. Otherwise, Nmap performs a full TCP handshake
    • Default port: 80 (change with -PS21 or -PS21-25)
  • TCP ACK ping: sudo nmap -PA -sn <target>
    • Requires root privileges
    • Default port: 80 (change with -PA21 or -PA21-25)
  • UDP ping: sudo nmap -PU -sn <target>

üí° Host discovery can be skipped using the -Pn (or -PN) option. This causes Nmap to proceed directly to the port scanning phase.

Host discovery without Nmap

If nmap is not available, different techniques can be used to discover hosts.

1. Using ARP requests (layer 2)

  • Check the ARP table (hosts must have communicated recently): arp -a
    • If the arp command is not available: cat /proc/net/arp
  • Use arping: for i in {1..254}; do arping -c 1 172.19.0.$i -I eth0 | grep "bytes from"; done

2. Using ICMP requests (layer 3)

  • Using fping: fping -a -g <start_IP> <end_IP> 2>/dev/null
  • Using the ping command (source):
    • ‚ùó May be ignored by Windows hosts. Use ARP requests instead.
# /24
for i in {1..254}; do (ping -c 1 172.19.0.${i} | grep "bytes from" | grep -v "Unreachable" &); done;
# /16
for i in {0..255}; do for j in {1..254}; do (ping -c 1 10.129.${i}.${j} | grep "bytes from" | grep -v "Unreachable" &); done; done;

3. Port scanning (TCP and UDP scan)

  • Simple TCP scan: nmap -sT <target> or TCP SYN scan (root user): sudo nmap -sS <target>. Additional options:
    • -F: fast mode, scans the 100 most common ports instead of 1000
    • -r: scan ports in sequential order instead of random order (useful when target is booting and services appear progressively, ‚ùóbut less evasive, IDS can detect sequential scans)
    • üí° If nmap is not available use /dev/tcp/<host>/<port> (source)
for PORT in {0..1000}; do timeout 1 bash -c "</dev/tcp/172.19.0.1/$PORT &>/dev/null" 2>/dev/null && echo "port $PORT is open"; done
  • Scan multiple targets (from an input file): nmap -iL list_of_hosts.txt
    • üí° Performs host discovery and TCP port scanning on each target (-sS if run as root, -sT otherwise)

Then we can follow logical enumeration steps (not strict nmap phases):

  • Detect service versions and run default scripts: nmap -sC -sV <target>

    • -sC : runs default NSE scripts - equivalent to --script=default and --script=safe,intrusive ‚ùó
    • üí° Combine with -sV (some scripts rely on version detection)
    • Optional: -sV --version-intensity 0-9 ‚Üí 0 = fastest, least accurate 9 = slowest, most complete

  • OS detection:

    • OS detection: sudo nmap -sS -O <target> - returns results only if nmap is confident
    • OS guessing: if no result, use --osscan-guess (enables OS guessing)

  • Run vulnerability detection scripts: nmap --script vuln -sV <target>

    • üí° Recommended to run -sC (light) and vuln (aggressive) scans separately
    • üí° Requires version detection (-sV)

Optionally, nmap -A groups all these options (-sC,-sV,-O,--traceroute), but is better to separate the different step (and add --traceroute if needed).

  • Then, perform targeted service enumeration using NSE scripts, e.g.: nmap --script smb-os-discovery -p445 <target>

  • Scan ALL TCP ports: because default scan covers only 1000 common ports (may miss open ports):

    • Using nmap -p- <target> then -sC -sV -p <open_ports> <target> (source)
    • üí° Use -T2 if results are inconsistent or incomplete (e.g., firewall, unstable network or packet loss)

UDP Scan

  • Basic UDP scan: sudo nmap -sU <target>
    • Fast UDP scan 1: nmap -F -Pn -sU -sV --max-retries 2 --host-timeout 2m <target>
      • -sV : useful because many UDP services only respond to properly formatted packets. However, it slows down the scan (use --version-intensity 0 to reduce impact)
      • --host-timeout 2m : sets a maximum scan time of 2 minutes per host after which Nmap skips the host
      • --max-retries 2 : limits retransmissions to 2 attempts
    • Fast UDP scan 2 (using rtt-timeout options): nmap -sU --min-rtt-timeout 50ms --max-rtt-timeout 200ms --initial-rtt-timeout 100ms <target>

‚ùó Scanning all UDP ports is not recommended, as it is extremely slow even with optimizations

4. Advanced ports scan and useful options

Advanced scan techniques

The following scan types are alternatives to standard TCP scans for identifying port states (except TCP ACK scan, which detects firewall rules). They do not complete the TCP three-way handshake :

  • TCP Null scan - all flags set to 0: sudo nmap -sN <target>
  • TCP FIN scan - FIN flag set: sudo nmap -sF <target>
  • TCP Xmas scan - FIN, PSH, URG flags set: sudo nmap -sX <target>
  • TCP Maimon scan - old BSD only: sudo nmap -sM <target>
  • TCP ACK scan - detect firewall presence: sudo nmap -sA <target>
  • TCP Window scan - window size check: sudo nmap -sW <target>

Scan tuning and evasion options

  • Timing option - controls packet rate : -T3 by default
    • T1 or T2 may help avoid IDS/IPS detection
    • Manual delay : nmap --scan-delay 2s IP_address
    • Packet rate options: --min-rate 15 (<= 15 packets/sec) and --max-rate 15 (>= 15 packets/sec)
    • Parallelization - control parallel probes: --min-parallelism=512
    • Disable reverse DNS resolution: -n

Debugging and analysis options

  • --reason: shows why Nmap made each decision
  • -v ‚Üí -vvvv: increases verbosity

Conclusion

In conclusion, effective port scanning with Nmap requires a structured approach, applying each phase progressively to ensure accurate and reliable enumeration.

Although this approach provides an essential foundation, it can be further enhanced by more advanced techniques, including in-depth analysis of results, advanced use of NSE scripts, and evasion techniques.
